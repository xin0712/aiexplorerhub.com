## 我的真实协作流程（可复用）

我这次用的是一个很稳定的协作循环：

**目标 -> 执行 -> 截图 -> 决策 -> 继续执行**

具体是这样：

- 我负责：给目标、做业务决策、确认选项
- Cursor 负责：查文档、给具体命令、解释报错、收敛路径

你可以把它理解为：  
我不是在“自己部署”，而是在“带着一个 24 小时在线的技术同事一起部署”。

### 3) 微信接入：我只做选择题，Cursor做判断题

- 我不需要记每个配置项含义
- 我只要截图当前页面，问“选哪个？”

比如这几类决策：

- 选 `Weixin / WeChat` 还是 `WeCom`

### 4) 真问题出现场：不是“有没有报错”，而是“怎么继续”

这次最真实的部分，是连续碰到问题：

- SSL 登录失败

传统教程会在这里断掉。  
这次协作里，我做的是“持续贴输出”，Cursor 做的是“持续给下一步”。

最后问题都被逐个消化，而不是变成“我先去研究两天”。

## 我最后的配置选择（真实结果）

为了保证稳定和可复制，我最后定的是：

- 微信路线：`Weixin / WeChat`（个人微信）
- 私聊：允许全部（先跑通）
- 群聊：禁用（减少 iLink 限制带来的不确定性）
- 服务器：海外节点，Docker 常驻运行

这不是“最复杂”的方案，但这是“能稳定上线”的方案。

## 给想复现这条路的人 3 条建议

1. **先说目标，不要一上来问命令**  
   例如：“我要部署Hermes到本地，这是GitHub：https://github.com/nousresearch/hermes-agent”

2. **每到选择页就截图，不要硬猜**  
   你的截图越清晰，AI 的指导越准确。

3. **把报错原文完整贴出，不要只说‘报错了’**  
   很多问题只差一行关键日志。
