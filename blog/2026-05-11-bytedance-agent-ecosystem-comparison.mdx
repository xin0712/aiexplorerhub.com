2026 年，Agent 已经不再只是一个 AI 热词。

从 OpenAI、Anthropic 到字节、阿里、腾讯，各家都开始从“聊天机器人”走向真正的 AI Agent 平台。

而在字节体系内，实际上已经形成了三条不同的 Agent 产品路线：

- 飞书妙搭（Feishu Miaoda）
- 飞书 Aily
- ArkClaw 企业版

很多人第一次看到时会困惑：

- 为什么字节会同时做三款 Agent 产品？
- 它们之间到底是什么关系？
- 谁更适合企业？谁更适合开发者？谁才是真正的“企业级 Agent”？

这篇文章，我们就从产品定位、底层能力、适用场景、技术架构几个维度，深度拆解字节目前的 Agent 布局。

## 一、字节为什么会同时做三种 Agent 产品？

因为 Agent 其实对应的是三个完全不同的问题。

### 第一类问题：企业怎么“使用 AI”？

对应产品：飞书 Aily。

Aily 的核心定位并不是“开发平台”，而是企业里的 AI 员工。它更像一个长期驻扎在飞书里的智能伙伴。

它知道：

- 你的聊天上下文
- 你的文档
- 你的会议
- 你的工作流
- 你的团队协作关系

所以 Aily 的重点不是“生成页面”，而是：

- 帮员工完成工作
- 帮团队协同
- 帮企业沉淀知识
- 帮组织自动化流程

本质上，Aily 更接近“企业协作 Agent”。

### 第二类问题：企业怎么“快速做 AI 应用”？

对应产品：飞书妙搭。

妙搭的思路非常明确：它并不想把 AI 做成一个聊天助手，而是用 AI 帮你生成应用。

一句话描述妙搭：**不会写代码的人，也能做系统。**

你只需要：

- 用自然语言描述需求
- 上传截图
- 说清楚逻辑

AI 就会：

- 自动生成前端页面
- 自动生成数据库
- 自动生成接口
- 自动生成后台逻辑

本质上，妙搭是“AI Native 的低代码平台”，也可以理解为“AI 产品经理 + AI 初级程序员”。  
它解决的是企业内部大量“小系统没人做”的问题，比如数据看板、CRM、营销活动页、内部审批系统、项目管理工具、AI 工作台。

### 第三类问题：企业怎么“真正部署 AI Agent”？

对应产品：ArkClaw 企业版。

ArkClaw 企业版和前两个不是一个层级，它更像企业级 Agent 基础设施。

它解决的是：

- 企业怎么大规模部署 Agent
- 怎么保证安全
- 怎么做权限管理
- 怎么控制 Token 成本
- 怎么做私有化部署
- 怎么让 Agent 接企业微信/钉钉/飞书
- 怎么做 VPC 隔离
- 怎么做审计日志

所以 ArkClaw 更偏向 **Agent Operating System（Agent 操作系统）**，它不只是一个聊天机器人，而是企业级 AI Runtime。

## 二、三款产品最大的区别是什么？

很多人会误以为它们只是 UI 不一样，但真正差异在于：**Agent 所在层级不同**。

### 飞书妙搭：AI Coding 层

妙搭的核心是“生成应用”，重点在：

- AI 写代码
- AI 搭页面
- AI 搭数据库
- AI 搭业务系统

它更像 Cursor + Replit + 低代码平台的结合体。  
所以妙搭本质不是“长期 Agent”，而是 AI 开发工具。

### 飞书 Aily：协作 Agent 层

Aily 的核心是“让 AI 成为员工协作者”，并长期驻扎在飞书里。

具备能力包括：

- 上下文记忆
- 群聊理解
- 工作流触发
- 文档分析
- 主动任务能力

所以它更像企业版 ChatGPT + 企业工作流 Agent。

### ArkClaw 企业版：Agent Runtime 层

ArkClaw 更底层，本质是 Agent 的运行环境。

重点在：

- Skill 调度
- 多 Agent 协同
- 云端运行
- 安全隔离
- Token 管理
- 企业级权限体系
- 长期记忆
- 多 IM 接入

所以 ArkClaw 更接近 LangChain / AutoGen / OpenClaw / Agent OS 这一层。

## 三、从企业视角，应该怎么选？

这取决于你到底想解决什么问题。

### 如果你想“快速做一个 AI 应用”

选：飞书妙搭

适合：

- 产品经理
- 运营
- 小团队
- 创业团队

尤其适合 MVP 验证。

### 如果你想“让 AI 提升员工效率”

选：飞书 Aily

适合：

- 飞书深度用户
- 中大型企业
- 日常协作场景

比如 AI 财务助手、AI 销售助手、AI 项目经理、AI 数据分析助手。

### 如果你想“真正做企业级 Agent 平台”

选：ArkClaw 企业版

适合：

- 金融
- 医疗
- 政务
- 大型互联网公司
- 多部门复杂组织

它解决的是企业级 AI 架构问题。

## 四、字节的 Agent 战略，其实是“三层结构”

把三款产品放在一起看，字节实际在构建一套完整 Agent 生态。

| 层级 | 产品 | 作用 |
| --- | --- | --- |
| 应用生成层 | 飞书妙搭 | 用 AI 快速做系统 |
| 协作智能层 | 飞书 Aily | 用 AI 提升组织效率 |
| Agent 基础设施层 | ArkClaw 企业版 | 运行和管理企业 Agent |

这三者不是竞争关系，而是上下游关系。未来甚至可能出现：

- 妙搭生成应用
- Aily 调用应用
- ArkClaw 托管运行

最终形成完整的 Agent 闭环。

---

## 五、Agent 时代真正的竞争，不只是模型

过去 AI 的竞争是“谁模型更强”，现在逐渐变成“谁更懂企业工作流”。

真正重要的能力开始变成：

- 记忆
- 权限
- 工作流
- Tool Calling
- Agent 协同
- 长期运行
- 企业知识
- 安全隔离

而这恰恰也是字节现在重点布局的方向。

从这个角度看，飞书妙搭、Aily 和 ArkClaw 并不是三个孤立产品，而是字节 Agent 战略里的三块关键拼图。
