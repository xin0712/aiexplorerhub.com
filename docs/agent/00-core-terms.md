这个页面用于持续沉淀 Agent 相关术语。

| 序号 | 名词 | 解释 | 例子 |
| --- | --- | --- | --- |
| 1 | Runtime | Agent 的运行时执行层，负责调度模型调用、工具调用、状态管理、超时重试与任务恢复，保证任务在真实系统中可控执行。 | 当用户发起“生成周报并发送邮件”时，Runtime 会按步骤调用检索、总结、邮件发送等能力，并记录每一步状态。 |
| 2 | AI Native | 指从产品设计到技术架构都以 AI 为中心构建的系统形态，而不是在传统系统上简单叠加 AI 功能。 | 一个 AI Native 的知识库产品，会默认用语义检索、Agent 协作和自动总结驱动核心流程，而不是仅加一个聊天入口。 |
| 3 | VPC隔离 | 通过虚拟私有云（VPC）将企业网络环境与公网隔离，限制 Agent、模型服务和数据资源的访问边界，以提升安全性与合规性。 | 企业把 Agent Runtime 部署在专属 VPC 中，只允许访问白名单数据库和内部 API，外部网络无法直接访问核心数据。 |
| 4 | 审计日志 | 对 Agent 的关键操作进行可追踪记录的日志机制，通常包含操作者、时间、动作、目标资源与结果，用于安全审计与问题追溯。 | 当某个 Agent 读取了敏感文档并调用外部工具时，系统会在审计日志中记录“谁在何时对哪些数据做了什么操作”。 |
| 5 | PR | Pull Request 的缩写，指在 Git 平台上提交的一组待合并代码变更，用于代码评审、讨论和协作合并。 | 开发者完成功能后创建一个 PR，团队成员在 PR 里 review 代码、提出修改意见，确认后再合并到主分支。 |
| 6 | Prototype | 原型版本，指用于快速验证需求、交互或技术可行性的早期产品形态，强调“先验证再完善”。 | 团队先用一周做出 AI 助手 Prototype，验证用户是否真的需要“自动整理日报”功能，再决定是否正式开发。 |
| 7 | CIO | Chief Information Officer 的缩写，通常指企业中的首席信息官，负责信息化战略、技术治理与数字化能力建设。 | 在企业推进 Agent 平台时，CIO 需要统筹数据安全、系统集成、预算投入和技术路线。 |
| 8 | Tool Calling（Function Calling） | 指模型在任务执行过程中按结构化参数调用外部工具（如 API、数据库、搜索、脚本）的能力，是 Agent 从“会回答”走向“会执行”的关键机制。 | 用户让 Agent 分析昨日投放数据时，Agent 通过 Tool Calling 自动调用 SQL 工具查询数据，再生成结论和建议。 |
| 9 | Human-in-the-Loop | 指在 AI 或 Agent 的决策与执行流程中引入人工参与，通过人工审核、确认或干预来提升准确性、安全性与可控性。 | 在自动回复客户邮件前，系统先生成草稿并由人工一键确认后再发送，这就是典型的 Human-in-the-Loop 流程。 |
| 10 | Harness | 指承载和托管 Agent 系统运行的上层框架，负责把 Runtime、Controller、Planner、Memory、工具连接与治理机制组织成可持续运行的整体。 | 在企业级多智能体系统中，Harness 负责统一管理任务生命周期、权限边界、监控告警与发布策略，让多个 Agent 能稳定协作。 |
| 11 | MCP（Model Context Protocol） | 一种让模型以统一协议连接外部工具、数据源与上下文资源的标准接口，帮助 Agent 在多工具场景下实现可扩展、可治理的集成能力。 | 团队把 GitHub、Postgres、飞书等能力封装成 MCP Server 后，Agent 可以通过统一方式发现并调用这些工具，而不需要分别适配每个系统。 |
